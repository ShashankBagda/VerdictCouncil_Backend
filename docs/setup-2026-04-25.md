# GitHub setup — Sprint 4 4.D3 eval gate

Authored 2026-04-25 alongside `.github/workflows/eval.yml`.

## Required repo secrets (4.D3.2 — human action)

The eval workflow needs these in **Settings → Secrets and variables →
Actions → Secrets**:

| Secret | Source | Used by |
|---|---|---|
| `LANGSMITH_API_KEY` | smith.langchain.com → Settings → API Keys | `tests/eval/run_eval.py`, `tests/eval/compare_to_baseline.py` |
| `OPENAI_API_KEY` | platform.openai.com → API Keys | `--mode graph` only (not the default stub run) |

The legacy `MLFLOW_TRACKING_URI` and `COHERE_API_KEY` are **not**
required — both were retired in earlier sprints.

## Required repo variable (4.D3.1)

| Variable | Source | Default behaviour if unset |
|---|---|---|
| `EVAL_BASELINE_EXPERIMENT` | LangSmith experiment name from Sprint 3 3.D1.4 | Workflow runs in observe-only mode (warning emitted, no gate) |

The variable lives under **Settings → Secrets and variables → Actions
→ Variables** (NOT secrets — it's a non-sensitive identifier). Bumping
the baseline after a sanctioned model change is a one-line variable
update; no workflow edit needed.

## Branch protection (4.D3.3 — human action)

Configure under **Settings → Branches → Branch protection rules** for
the `main`, `release/*`, and `development` branches:

- ☑ **Require a pull request before merging**
- ☑ **Require status checks to pass before merging**:
  - `Lint`
  - `Unit Tests`
  - `Security`
  - `Eval Gate` ← from this workflow
  - `Docker Build` (if applicable)
- ☑ **Require conversation resolution before merging**
- ☐ **Require signed commits** — optional; project does not currently
  enforce.

CODEOWNERS is intentionally **not** used (per user 2026-04-25
decision); the eval gate stands as the technical bar for prompt /
pipeline changes, branch-protection requires human review.

## Bypassing the eval gate

Apply the `eval/skip-regression` label to a PR to skip the
regression comparison. The eval **still runs** (so the numbers appear
on the PR), it just doesn't fail the check. Use cases:

- Sanctioned model upgrade — quality is intentionally moving
- Dataset rewrite — the comparison would compare apples to oranges
- Emergency hotfix where the eval baseline isn't representative

The label is dropped manually after merge so the next PR is gated
again. There is no auto-removal.

## Verifying the gate end-to-end (4.D3.4 — human action)

Smoke procedure to prove the gate can actually fail:

1. Open a PR that touches `prompts.py` and deliberately mis-aligns a
   prompt (e.g. delete an instruction the goldens depend on).
2. Confirm the workflow runs and the comparison step fails red.
3. Apply `eval/skip-regression`; re-run the workflow.
4. Confirm the comparison step short-circuits and the gate passes.
5. Drop the label; close the PR without merging.

This validates both the failure path and the bypass path. Capture the
two GitHub Actions run URLs in the runbook for posterity.
